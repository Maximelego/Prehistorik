Prehistoric

A small platformer like videogame.

Link to the GitLab project : https://gitlab.univ-lorraine.fr/marceli11u/prehistoric

Requirements : CMake verion 3.10 or above. A complete SDL2 library. [Not essential] : Doxygen software installed.

Available commands : "cmake CMakeLists.txt" : Creates the makefile for the project. "make" : Creates an launchable version of the project. "make clean" : Removes all compiled files. "make doxygen" : Generates an HTML documentation of the Project.

"./Prehistoric" : after using the commands "cmake CMakeLists.txt" & "make", launches the program.

Please refer to CHANGELOG.txt to see if the project is compilable.

Also, make sure that the SL2, SDL-TTF and SDL-MIXER libraries are correctly installed.

Here's a list of commands to make sure they are all there and up to date.

sudo apt-get install libsdl-image1.2 libsdl-image1.2-dev (for the Image Library) sudo apt-get install libsdl-ttf2.0-0 libsdl-ttf2.0-dev (for the ttf library) sudo apt-get install libsdl-mixer.1.2 libsdl-mixer1.2-dev (for the ttf library)

//-------------—LEVEL CREATION :-------------—//

You can manually add levels to the game !

In order to do this, you will need, however, to follow some guideline :

A level is organized as it follows :

  • A level layer is 100 pixels thick and there are 6 layers in one level.
  • The highest layer of the level should not take any platforms (It is not impossible to do so, but the player will not be visible in the screen).
  • All elements must be organized by following the level's columns (By default, there are 100 columns available to use, you can still make the level shorter.).
  • A level column is 100 pixel thick.

To add a level to the game, simply create a .level file in the Ressources/Levels folder. Be rassured, if the level is not correctly made, it will not crash the game, but the game will not add it to it's level list.

You can create as many levels as you want.

The Syntax is the following : 

- To define a Platform :
            P X Y Z     X : X Column. | Y : Y Layer. | Z : Platform thickness (must be between 1 and 5 !)
  • To define a Mob : M T X Y T : Mob Type (look for thes available below.) | X : X Column. | Y : Y Layer.

Usable char : 

//ELEMENTS.//
P : defines a platform
M : defines a mob

//MOB TYPES.//
0 : defines a crocodine mob.
1 : defines a Dinosaurus mob.
2 : defines a Scorpion mob.


exemple :
M 0 3 2                 -> Defines a crocodile mob on the 3 column and 2nd layer.
P 1 2 1                 -> Defines a platform on the first column, 2nd layer and that is 1 column large.
P 3 1 1
P 4 1 1
P 2 2 1
